    Enterasys firmware yükseltme

    mofm

    2011-10-25 10:58

    Comments

    Her zaman güncel firmware'i kullanmak daha yararlıdır.Enterasys kenar switclerde firmware yükseltmek için küçük adımlar...

    Firmware yükseltme işlemine geçmeden önce bir tane TFTP sunucusunu hazır etmeniz lazım.Yoksa XMODEM ile saatlerce beklemeniz lazım :) (Tabi switch üzerindeki tüm "Operation Code"ları(firmware) silerseniz mecburen XMODEM ile yüklemek ve beklemek zorundasınız.)

    1.Adım:

    Eğer switch üzerinde birden fazla firmware varsa,büyük ihtimal yeni firmware yüklemeye yer olmayacaktır.O yüzden uer açmak ilk olarak kullanılmayan eski firmware silelim : # dir       (bu komutla eski firmwareleri listeleyin.)

    Images:

    Filename:      c2-series_05.00.75 Version:       05.00.75 Size:          7677952 (bytes) Date:          Tue Aug  7 09:41:04 2007 CheckSum:      4d76baca7158f071f9ef1245d22c5466 Compatibility: C2G124-24, C2G124-48, C2H124-48, C2G124-48P, C2H124-48P C2K122-24, C2G170-24, C2G134-24P, C3G124-24, C3G124-24P C3G124-48, C3G124-48P

    Filename:      c2-series_05.02.07.0006 (Active) (Boot) Version:       05.02.07.0006 Size:          9479168 (bytes) Date:          Sat Oct 10 15:22:25 2009 CheckSum:      06bebc6af033e7e27a0327f04c991164 Compatibility: C2G124-24, C2G124-48, C2H124-48, C2G124-48P, C2H124-48P C2K122-24, C2G170-24, C2G134-24P, C3G124-24, C3G124-24P C3G124-48, C3G124-48P, C3K122-24, C3K122-24P, C3K172-24

    Files:                           Size ================================ ======== configs: 07102011                         12389 logs: current.log                      167441

    delete c2-series_05.00.75  (siz kendi firmware numaranıza göre silin.)

    2.Adım: Bu adresten ürününüze uygun olan en son firmware'ı TFTP sunucunuza indirin. TFTP sunucumuza indirdiğimiz yeni firmware'ı switch'e aşağıdaki gibi yükleyin :

    copy tftp://192.168.1.12/c2-series_05.02.13.0003 system:image

    3.Adım: Yükleme başarı ile tamamlandıysa yeni firmware ile sistem başlatılır:

    set boot system c2-series_05.02.13.0003

    4.Adım:

    Switch yeniden başladıktan sonra yeni firmware kontrol edilir.

    show boot system

    Elite Proxy

    mofm

    2011-10-23 14:27

    Comments

    Squid tam bir derya ve bunlardan birkaç kısa not :

    * Squid proxy arkasındaki ip'ler hakkında bilgi vermemek için :

    /etc/squid/squid.conf içerisine ,

    forwarded_for off  ,-- satırını ekleyin.

    * Squid Proxy Server'ın versiyonu hakkında dışarıya bilgi vermemek için:

    httpd_suppress_version_string on  , -- satırını "/etc/squid/squid.conf" dosyanıza ekleyin.

    * Bazı proxy detect taraması yapan yazılımlardan ve buna benzer sitelerden ( http://checker.samair.ru/ , http://ip.my-proxy.com/ , vs) squid proxy server'ınızı gizlemek için aşağıdaki satırları konfigurasyon dosyanıza ekleyin :

    header_access Allow allow all header_access Authorization allow all header_access WWW-Authenticate allow all header_access Proxy-Authorization allow all header_access Proxy-Authenticate allow all header_access Cache-Control allow all header_access Content-Encoding allow all header_access Content-Length allow all header_access Content-Type allow all header_access Date allow all header_access Expires allow all header_access Host allow all header_access If-Modified-Since allow all header_access Last-Modified allow all header_access Location allow all header_access Pragma allow all header_access Accept allow all header_access Accept-Charset allow all header_access Accept-Encoding allow all header_access Accept-Language allow all header_access Content-Language allow all header_access Mime-Version allow all header_access Retry-After allow all header_access Title allow all header_access Connection allow all header_access Proxy-Connection allow all header_access All deny all

    Buraya küçük bir not düşeyim , 3.0 versiyon ve üzeri squid kullanıcıları "header_access" yerine "request_header_access" kullanmalıdır."header_access" 2.6 ve daha eski sürümler için geçerlidir.

    Kernelinizi Özgürleştirin!

    mofm

    2011-10-06 23:15

    Comments

    Birçok Linux dağıtımı kernelinde  özgür olmayan firmware (nonfree firmware) ile beraber geliyor.Kerneldeki bu kodlar özgür olmayan (nonfree) lisanslar kullanmakta ve bunlara özgür yazılım camiası “blobs.” demektedir.Özgürlüğüne düşkün kullanıcılar kernellerini bunlardan temizlemek istiyorsa Free Software Foundation(FSF) tarafından kernel sürümlerine göre yayınlanan scriptleri kullanabilirler.

    Bu linkte ( http://www.fsfla.org/svn/fsfla/software/linux-libre/scripts/ ) kernel sürümünüze uygun olan scriptleri kullanarak kernelinizi bunlardan temizleyebilirsiniz.

    Özgür günlerde , özgür kerneller kullanmanız dileğiyle :)
